一、算力账本：Test-Time Compute 的边际成本与 KV Cache 碎片化

推理成本模型必须从 FLOPs 转向内存访问。以 Llama 3 700B 参数模型为例，使用 FP8 精度，参数占用 700GB。KV Cache 内存占用公式：M_kv = 2 * n_layers * n_heads * d_head * n_tokens * b。其中 n_layers=80， n_heads=64， d_head=128， b=1 byte (FP8)。对于 100K 上下文（n_tokens=102400），单次推理的 KV Cache 为 2*80*64*128*102400 ≈ 1.34e11 字节 ≈ 125GB。优化后（分组查询注意力）仍需约 200MB。物理现实是：HBM3e 80GB 芯片在服务 1000 个并发请求时，仅 KV Cache 需求即为 200GB，超出物理容量 150%。系统必须将 KV Cache 换出至 DRAM，通过 PCIe 5.0（64GB/s）或 NVLink 4（900GB/s）搬运。计算有效算力利用率 η = (理论 TFLOPS) / (1 + (内存延迟周期 / 计算周期))。当 KV Cache 未命中率超过 30% 时，A100/H100 的 312 TFLOPS FP8 性能降至 ≤ 11.5 TFLOPS。这意味着每 1 美元推理收入中，有 0.86 美元用于支付数据搬运的电费与延迟成本。

二、内存墙解剖：HBM3e 的带宽、功耗与封装死锁

HBM3e 的 6.4TB/s 带宽由 1024-bit 接口与 3.2Gbps/pin 速率实现。物理瓶颈在于硅通孔（TSV）密度与热耗散。当前 12-hi 堆叠的 TSV 密度约为 10k/mm²，每层逻辑 die 的功耗为 2W。12 层堆叠总热功耗密度超过 1kW/cm²，必须通过 55μm 微凸点与硅中介层散热。TSV 良率公式 Y_tsv = (1 - defect_density)^(n_tsv * n_layers)。当缺陷密度为 0.01/cm²，单芯片 TSV 数量为 5 万时，12 层堆叠良率 Y_tsv ≈ 0.54。这直接导致合格 HBM3e 芯片成本比标称高 85%。带宽提升路径依赖 TSV 速率提升至 5Gbps/pin 与堆叠层数增至 16-hi。但热力学方程 ΔT = P * R_th 表明，在 R_th（热阻）不变情况下，功耗 P 增加 33% 将使结温 T_j 超过 105°C 的可靠性阈值，引发每秒 0.1% 的位错误率。因此，2026 年后 HBM 每代带宽提升将低于 25%，单位比特成本下降曲线趋于平坦。

三、互联拓扑：硅光子的成本方程与系统级瓶颈

铜互联在 5mm 以上距离的能耗超过 20pJ/bit，硅光互联理论极限为 4pJ/bit。但该数值未计入激光器、调制器与光电转换的固定功耗。基于 400G 硅光引擎的实测数据：56Gbaud 硅基调制器（MZM）的 Vπ * L 乘积为 2.2 V·cm，驱动电压摆幅需 3V，导致每通道功耗 1.5pJ/bit。外接 III-V 族激光器（贴装或外延）的 wall-plug 效率仅 15%，每通道额外增加 2.8pJ/bit。系统总能耗为 1.5 + 2.8 + 0.7（TIA）≈ 5.0pJ/bit。成本方面，8英寸硅光晶圆与 12英寸 CMOS 逻辑晶圆工艺不兼容，需要 2.5D/3D 异质集成，每比特成本为 0.12 美元/GB/s，是 NVLink 4 铜互联方案的 8.3 倍。在 1024 个 GPU 的全连接拓扑中，光互联虽可降低 40% 的延迟，但资本支出（CapEx）增加 300%，投资回收期延长至 7 年，不符合对冲基金 <3 年的持有周期要求。

四、投资结论：物理定律下的资本开支 ROI 拐点

综合上述参数：设算力需求年增长 80%（符合当前训练数据增长），而单位算力有效性能因内存墙年增长仅 20%。定义算力效率衰减系数 α = (有效 TFLOPS / 标称 TFLOPS)。当前 α=0.37，模型推演至 2027 年，α 将降至 0.12。这意味着维持相同有效算力输出的资本开支 CAGR 需达到 67%，远超云服务商 35% 的营收增长率。现金流折现模型（WACC=12%）显示，2024 年后投入的万卡集群，其净现值（NPV）将在 2028 年转为负值。结论：1. 做多存算一体、近内存计算等绕过内存墙的架构初创公司（如 Groq, SambaNova）。2. 做空严重依赖线性扩展 HBM 带宽与 GPU 堆叠的纯硬件厂商。3. 2026 年是资本开支边际收益归零的临界点，需提前 18 个月调整头寸。
