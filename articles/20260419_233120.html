一、算力账本：Test-Time Compute 的 TCO 陷阱与 HBM 热崩溃

市场关注点错误地集中于训练算力（FLOPs），而推理阶段的算力消耗（Test-Time Compute）将定义商业模型的可行性。对于一个参数量为 N 的模型，单次前向传播的浮点运算次数约为 2N。若每日处理 Q 次查询，每次查询平均令牌长度为 L，则每日总推理算力需求为 2NQL FLOPs。

以 1.76T 参数模型（GPT-4 规模）为例，N = 1.76e12。假设 Q = 1e9（日十亿次查询），L = 1000 令牌，则日推理算力需求为 3.52e24 FLOPs。这需要约 4082 块峰值算力为 1000 TFLOPS 的 H100 GPU 满负荷运行 1 秒，或等效于 47 张 H100 持续运行一天。电费成本（按 0.1美元/kWh，GPU 功耗 700W）每日超过 7.9 万美元，年化 2885 万美元。这仅是计算功耗，未包含内存系统功耗。

HBM 内存墙的核心是带宽与功耗的平方律关系。HBM3e 目标带宽 1.2 TB/s，其 I/O 功耗遵循公式 P = α * C * V² * f，其中 α 为活动因子，C 为负载电容。在 3 pJ/bit 能效下，仅维持此带宽的 I/O 功耗即为 1.2e12 * 3e-12 = 3.6 W。然而，12-Hi 堆叠的垂直通孔（TSV）密度导致热流密度激增。硅中介层的热导率约为 150 W/(m·K)，TSV 阵列区域的热密度可超过 100 W/mm²，引发局部热点超过 110°C，导致晶体管迁移率下降与时序违例。良率模型显示，每增加一个堆叠层，良率乘数下降 0.85。12-Hi 堆叠的复合良率低于 30%，致使每 GB HBM 成本从 20 美元飙升至 28 美元以上。

二、互联拓扑：硅光的经济性谬误与 NVLink 拥塞

硅光互联被宣传为突破电气互联距离-带宽瓶颈的解决方案。其物理极限由光电转换效率决定。当前最先进硅光调制器的能耗约为 5 pJ/bit，而先进 SerDes 在 2 米距离内的能耗已降至 2 pJ/bit 以下。因此，在机架内（<2m）的 GPU 间互联，采用硅光将增加 150% 的互联能耗。延迟方面，光电转换引入额外 10 ns 延迟，对于需要亚微秒级同步的 All-Reduce 操作是致命缺陷。

真正的瓶颈在于多智能体并发推理场景下的 NVLink 拥塞。假设一个 8-GPU 节点通过 NVLink 4.0（900 GB/s 双向带宽）全互联。当服务多个长上下文请求时，KV Cache 需要在 GPU 间动态迁移。每个 100K 上下文的 FP8 KV Cache 大小约为 2 * (100,000 * 12,288) = 2.46 GB（假设头维度 128）。若 4 个此类请求并发，需迁移约 9.84 GB 数据。即使以 900 GB/s 理论带宽迁移，也需 11 ms。在此期间，NVLink 总线被占满，其他计算任务停滞。总线利用率的排队模型显示，当请求到达率 λ 超过总线服务率 μ 的 60%，平均排队延迟呈指数增长，有效算力利用率降至 70% 以下。

三、内存架构：KV Cache 碎片化与 SRAM 的面积惩罚

自回归解码导致 KV Cache 随生成令牌数线性增长。在多租户、多请求的云服务环境中，KV Cache 的分配与释放是随机的，产生严重的内存碎片。定义内存利用率 η = (有效 KV Cache 总大小) / (已分配 HBM 总大小)。模拟显示，对于包含 1000 个并发会话的服务，η 随时间衰减至 0.45-0.55 区间。这意味着 80GB HBM 中，仅有 36-44GB 用于有效数据存储，其余被碎片占用。

解决方案之一是增大片上 SRAM 以缓存活跃的 KV Cache。然而，SRAM 面积成本极高。在 5nm 工艺下，1 MB SRAM 面积约为 0.02 mm²，能效为 0.5 pJ/bit。若为每个注意力头配备 128 KB 专用 SRAM（例如 128 头需 16 MB），仅 SRAM 面积就达 0.32 mm²。对于一个 800 mm² 的大芯片，这占用 0.04% 的面积，但考虑到芯片上通常有数十个此类模块，总面积占比可超过 15%，直接挤占计算单元（Tensor Core）的空间，导致峰值算力下降。面积-性能的帕累托前沿表明，SRAM 面积超过总芯片面积的 20% 时，每增加 1% 的 SRAM，峰值 TFLOPS 下降 0.7%。

四、投资结论：物理约束下的资本重新定价

1. **HBM 供应商估值过高**：HBM 的 thermal-limited scaling 将在 2025-2026 年触顶。每 GB 成本下降曲线已趋于平缓，甚至逆转。依赖 HBM 密度增长叙事的企业面临估值下调风险。
2. **光学互联公司短期不具规模经济**：在数据中心机架内尺度，硅光互联的功耗与成本劣势明显。投资应聚焦于 >10 米的长距离互联场景，该市场总量仅为机架内互联的 15%。
3. **架构创新溢价凸显**：具备 KV Cache 压缩、碎片整理或异构内存（HBM+大容量DRAM）管理能力的硬件/软件公司，能直接提升有效内存利用率 η，其价值被低估。η 每提升 0.1，对应单卡有效吞吐量提升 22%，TCO 下降 18%。
4. **推理专用芯片（Inference ASIC）的必然性**：通用 GPU 为训练优化的宽向量单元在推理时利用率不足 40%。定制化 ASIC 通过增大 SRAM 占比、优化数据流，可将能效（TOPS/W）提升 3-5 倍，这是突破 Test-Time Compute TCO 陷阱的唯一物理路径。

算力市场的下一阶段竞争，将从晶体管密度竞赛转向架构与封装层面的热、电、数据流协同优化。违反物理定律的资本叙事将在此过程中被清算。
