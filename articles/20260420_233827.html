一、算力账本：Test-Time Compute 的物理极限与成本函数

定义推理成本函数 C_infer = (P_compute * T_compute + P_memory * T_memory) * Cost_per_Watt。当前架构下，T_memory 主导 T_compute。

以 Llama 3 70B 模型为例，参数 70B，使用 FP8 量化（1 byte/param）。KV Cache 内存占用公式：M_kv = 2 * n_layers * n_heads * d_head * n_ctx * precision。代入：n_layers=80, n_heads=64, d_head=128, n_ctx=128K, precision=1 byte。计算得 M_kv ≈ 2 * 80 * 64 * 128 * 131072 * 1 ≈ 1.7GB。此值为单序列。1000 并发请求下，总 KV Cache 需求为 1.7TB，远超单 GPU HBM 容量。系统必须采用动态换入换出策略。

每次换出的时间惩罚：从 HBM 通过 NVLink（900GB/s）迁移 1GB KV Cache 至 CPU DRAM（100GB/s）需 11ms。在此期间，计算单元闲置。假设 10% 的 KV Cache 需要换出，则单次推理延迟增加 110ms。对于要求 100ms 尾延迟的在线服务，此惩罚不可接受。因此，实际部署中，并发数被强制限制在 HBM 容量除以单序列 KV Cache 大小的商，即 ≈ 80GB / 1.7GB ≈ 47。这直接粉碎了“单卡千并发”的营销叙事。

二、架构解剖：HBM 内存墙与互联拓扑的拥塞模型

HBM3e 的 1.2TB/s 带宽由 1024 个数据引脚（每通道 32-bit，共 32 通道）在 3.2Gbps 速率下实现。物理限制：信号完整性要求 TSV（硅通孔）长度与直径比小于 10:1，当前 8-Hi 堆叠高度约 720μm，TSV 直径需 >72μm。这导致单位面积内可布设的 TSV 数量存在上限，限制了通道数扩展。

热模型：每个 HBM 堆栈功耗约 15W。在 8-Hi 堆叠中，热量通过 TSV 和微凸点垂直传导。热阻公式 R_θ = t / (k * A)，其中 t 为硅中介层厚度，k 为硅热导率（150 W/mK），A 为传热面积。计算显示，中心逻辑 die 与底部封装的热阻导致结温与外壳温差可达 40°C。当环境温度 45°C 时，结温达 85°C，载流子迁移率下降，TSV 电阻上升，有效数据速率衰减 30%。

多 GPU NVLink 拓扑在 KV Cache 碎片化场景下失效。假设 8 台 GPU 以全连接拓扑互联，每台 GPU 持有部分 KV Cache。生成下一个 token 时，需要全局注意力分数，引发 all-to-all 通信。NVLink 4.0 每方向 900GB/s，但 8 节点全连接需要 56 条链路，实际布线不可能。采用 NVIDIA 的 NVSwitch 树状拓扑，根节点带宽成为瓶颈。模拟显示，在 70% 负载下，根节点拥塞导致延迟从 0.5μs 激增至 5μs。

三、颠覆路径：硅光互联与近存计算的经济性临界点

硅光互联的宣称优势：能耗 100 fJ/bit（激光器+调制器+探测器），远低于铜互连的 2 pJ/bit（在 5mm 距离）。但此数据未计入 3D 封装内的耦合损耗。将光波导集成至硅中介层，需要 grating coupler 进行垂直耦合，插入损耗 3dB，即 50% 光功率损失。为补偿，需提高激光器功率，使系统级能耗升至 500 fJ/bit。

经济性分析：硅光芯片需要 III-V 族材料（如 InP）外延生长在硅上，良率低于 70%，单片成本是纯硅中介层的 3 倍。只有当互连密度要求超过 10 Tb/s/mm² 时，铜互连的串扰和功耗使其不可行，硅光方案才具备成本优势。此临界点预计在 2026 年后的 1nm 节点出现。

近存计算是更直接的解。将 SRAM 宏单元与计算单元在 3D 堆叠中垂直集成，通过 μBump 连接，带宽可达 4 TB/s/mm²，延迟 100ps。但 SRAM 面积效率低下：22nm SRAM 单元面积约 0.1μm²。存储 1MB 数据需要 8,388,608 个单元，面积约 0.84mm²。而同等容量的 HBM DRAM 单元面积仅 0.01mm²（得益于 1z nm DRAM 工艺）。近存计算用 80 倍面积代价换取 10 倍带宽提升，仅在特定高频小数据量算子（如 GELU、LayerNorm）上具有正 ROI。

结论推演：未来 24 个月，架构演进将分化为两条路径：1）牺牲通用性，采用定制化模型架构（如 Mamba）彻底规避 KV Cache，但损失任务泛化能力。2）接受更高的单次推理硬件成本，采用 Chiplet 设计将 HBM 与计算单元以 2.5D/3D 方式更紧密集成，将系统级有效带宽提升 2 倍，但芯片封装成本增加 50%。无论哪条路径，当前以通用 GPU 为核心的资本开支曲线将在 2027 年前触及 ROI 拐点。
