一、算力账本：Test-Time Compute的伪需求与HBM内存墙的物理锁死

定义推理算力需求为 R_inf = (2 * N_param * N_token * B) / T_latency。其中N_param为激活参数量，N_token为序列长度，B为批次大小，T_latency为可容忍延迟。当前市场假设T_latency恒定，盲目放大N_token与B。以Llama 3 70B模型、FP8精度为例，单次前向传播需70B * 2 Byte = 140GB参数读取。若服务上下文长度100K（N_token=100,000），批次B=8，则每请求需读取数据量：140GB + (100K * 70B * 2 Byte / 序列) ≈ 154GB。HBM3e单堆栈峰值带宽1024GB/s，理论服务时间150ms。然而，KV Cache（键值缓存）需持续驻留HBM。对于100K上下文，KV Cache体积为：2（K, V） * 层数（80） * 头数（64） * 头维度（128） * 100K * FP8 = 约163.84GB。此体积已超过单颗HBM3e堆栈容量（当前最大24GB）。实际部署需跨多颗HBM，通过NVLink互联。NVLink 4.0双向带宽900GB/s，但跨GPU访问KV Cache将引入额外1.5-2μs延迟。在8路GPU服务器中，尾部GPU访问头部GPU的KV Cache，通信延迟将占单Token生成延迟（约50ms）的30%以上。算力利用率η = 实际TFLOPS / 峰值TFLOPS。当通信开销占比超过30%，η降至35%以下。这意味着资本支出的65%用于填补通信缺口，而非有效计算。

二、架构解剖：KV Cache碎片化与片上SRAM的面积诅咒

Transformer解码阶段，KV Cache按（序列位置，注意力头，层）三维度分布。内存访问模式是随机的、细粒度的。HBM设计用于长突发（burst）传输，典型突发长度32-64字节。FP8精度的KV Cache单次访问仅1字节，导致HBM控制器效率暴跌。定义效率E = 有效传输数据量 / 总传输数据量。在顺序访问下E可达85%。在碎片化随机访问下，E实测为22%。这使实际可用带宽从1024GB/s降至225GB/s。解决方案是增大片上SRAM作为缓存。计算SRAM需求：每注意力头KV Cache体积 = 2 * 头维度（128） * 序列长度（100K） * FP8 = 256KB。单GPU需缓存80层*64头=5120个此类结构，总计1.31GB。采用台积电N3工艺，SRAM密度约20MB/mm²。存储1.31GB需面积65.5mm²。作为对比，NVIDIA H100 GPU的L2缓存为50MB，面积已超80mm²。将SRAM扩大26倍在物理上不可行：1）面积成本指数增长；2）静态功耗（约1pJ/bit）将使芯片总功耗突破3000W，散热系统崩溃。因此，KV Cache碎片化是内存子系统与计算单元解耦架构的固有缺陷，无法通过现有2.5D封装或工艺缩微解决。

三、互联拓扑：硅光子的能耗优势与延迟陷阱

硅光互联被宣传为突破“内存墙”的路径，其核心优势是能耗：电互联约15pJ/bit，硅光可降至3pJ/bit。然而，该指标忽略关键延迟分量。硅光链路包含：电信号→激光调制（>20ns）+ 波导传输（可忽略）+ 光电探测（>30ns）+ 信号恢复（>50ns）。总延迟>100ns。在数据并行训练中，此延迟可被批量掩盖。但在推理尤其是多智能体并发场景中，请求需在多个专家模型（MoE）或工具调用间跳转，产生大量细粒度、低批次的All-to-All通信。定义通信延迟T_comm = 启动延迟 + 传输延迟 + 同步延迟。电互联（如NVLink）启动延迟约10ns。当传输数据块小于1KB时，硅光的100ns固定延迟将主导T_comm。模拟显示，在128个智能体并发、平均消息大小512B的场景下，硅光系统的尾延迟（P99）是纯电互联系统的2.8倍。因此，硅光互联仅适用于流式大数据传输（如内存池化），而非推理服务的控制平面。资本押注硅光替代NVLink是方向性误判。

四、投资推论：物理瓶颈下的资本回报衰减曲线

基于以上推演，构建硬件投资回报模型ROI = (Throughput_per_Watt * Utilization) / (Cost_per_Chip * Power_Density)。当前架构（以H100为基准）的Utilization因内存墙限制锁定在35%。下一代芯片（如B100）即使算力提升3倍，若内存带宽仅提升1.5倍（至1.5TB/s），Utilization将进一步降至28%。因为算力增长与内存带宽增长失配，缺口Δ = (算力增长比 / 带宽增长比) = 3 / 1.5 = 2。Utilization衰减与Δ成反比。功耗方面，HBM3e堆栈从8层增至12层，TSV（硅通孔）密度从300/mm²提至500/mm²，热密度从0.8W/mm²升至1.2W/mm²，冷却成本增长非线性，预计增加70%。综合计算，未来18个月内，单次推理的硬件成本降幅将从历史曲线的年化50%骤降至15%以内。资本支出对推理服务规模的线性拉动效应将于2025年Q3前失效。突破路径并非堆叠更多HBM或Chiplets，而是架构层面重构：1）采用状态空间模型（SSM）等序列模型替代Transformer，从根本上消除KV Cache；2）推动存内计算（Compute-in-Memory）商业化，将权重与KV Cache访问移至3D NAND或MRAM阵列，即使其精度降至4-bit，但可提供>10TB/s的有效内存带宽。当前，这两条路径的研发资本投入不足总体的5%，存在巨大预期差。
